I've started learning programming because of VJ'ing and I thought it would be cool to project 100% real-time generated content. I've learnt first Winamp AVS and moved on to much more usable VVVV during this process. It helped me also to get into computer science and digital arts at the same time.

MD.BOT3


NU-SCHOOL

Recently rewritten framework with modern tools and concepts. This has separate Client VVVV instances simply wrapping a patch-switcher and some utilities serving a texture source, and a service VVVV instance rendering and mixing textures from client instances and broadcasting common data to them like audio and MIDI for instance.

MD.BOT2


MID-SCHOOL

First framework supporting both DX11 and DX9. It was a patch-switcher wrapper with some more advanced features. It worked in a service-less model so the output was rendered with a special patch wrapped in an md.bot2 instance. Because of that each instance had to gather data on its own, or in case of the audio and beat-sync they could toggle between master and slave modes.

MD.VIS / MD.BOT


OLD-SCHOOL

Remnants of my original window mixer for mixing AVS windows + a simple vvvv patch-switcher. First it was using the screenshot node (window capture) and later DX9EX shared textures when they became stable in VVVV.

WNDMXR


ANCIENT-SCHOOL

I've started VJ'ing with 2 or more instances of Winamp AVS. I've done considerable amounts of pretty fancy presets too and I mixed them together with VVVV. First this was only a slider and an output window but it grown out to be a full blown GUI.
