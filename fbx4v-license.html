FBX4V is free to be used only for non-commercial, educational, research and evaluation purposes. For Commercial applications please contact me at via fbx4v@mcro.de, and request a quote.

Note that FBX4V is still in a kinda "release candidate" phase. There are a few known issues and there are plans for features which would ease usage or save some performance.

A note about support: FBX4V has a general purpose geometry outlet which is compatible with every shader. Accessing Subset- Material- or Instance ID is an easy modification to any publicly available .fx or .gfx (GeomFX) shader (if they don't have already a similar pattern to use). In mp.dx there are shaders and a rendering pipeline which support those semantics but they are not part of FBX4V and they are open-source software under MIT license, which doesn't provide any guarantee or support. If I discover a bug in mp.dx, mp.fxh or mp.essentials which might render FBX4V or any part of it unusable, I will do my best to fix it asap. However if a bug-report turns out to be coming from afore-mentioned packs and further dependencies other than FBX4V, I can't provide the same support regarding those for non-commercial users. This doesn't mean I don't like bug-reports. I love them. In regards with FBX4V send them to fbx4v@mcro.de or in regards with other packs, the respective Github repository issue tracker.

A note about assets: provided assets are collected from various freely available sources with varying licenses. They are optionally provided here only for showcasing, educating and evaluating FBX4V. For the sake of simplicity they are all FORBIDDEN to be used anywhere else, including projects falling under the non-commercial definition.

Enjoy!
